# Definição (denotativa)
**Proeza** = desempenho **dificílimo** no **contexto dado**, com **superação mensurável** de limiares. Não requer ser único.  
**Proeza Singular (PS)** = proeza **+ unicidade verificável** (traços irrepetíveis).  
Diferença-chave: proeza = difícil; singularidade = único. PS ocorre quando ambos valem.

# Critérios objetivos (TEIA)
- **Dificuldade (D)**: supera p95 do histórico/benchmark relevante.  
- **Restrições (R)**: realiza sob limite duro (tempo, energia, memória, I/O) com prova.  
- **Impacto (Δ)**: ganho líquido mensurável (ex.: compressão, latência, throughput).  
- **Evidência (E)**: antes/depois + hashes + log com timestamp.

**Regra de decisão**  
- Se (D≥p95) ∧ (Δ≥LΔ) ∧ (E ok) ⇒ **Proeza**.  
- Se, além disso, **Unicidade (U=true)** por ≥3 traços diferenciais verificáveis ⇒ **Proeza Singular**.

# Métrica compacta
**Score_P = α·D_norm + β·Δ_norm + γ·R_norm** (α≥β≥γ; defaults α=0.5, β=0.3, γ=0.2).  
**δ-meta** = Δ_norm · (1 – Risco_op) · Irreversibilidade_norm.  
Etiqueta final: {“não proeza”, “proeza”, “proeza singular”} com Score_P e δ-meta.

# IO (tradutor TEIA→LLM)
Entrada: frase simbólica sua.  
Interpretação precisa: {D, R, Δ, E, U}.  
Saída: etiqueta + decisão binária + artefato-prova curto.

# Plano (1–3)
1) **Baseline**: fixar p95 por domínio (ex.: tempo de compressão, latência p95, ganho %).  
2) **Limiar**: definir LΔ e restrições válidas; escolher pesos {α,β,γ}.  
3) **Prova**: emitir **Proeza-Prova.json** (descricao, contexto, D, Δ, R, U, Score_P, δ-meta, hashes).
