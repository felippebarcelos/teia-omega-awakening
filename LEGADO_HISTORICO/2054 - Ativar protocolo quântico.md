- `multi_run.ps1` – lança vários nós em sequência, gerando uma semente aleatória para cada nome simbólico e chamando `run.ps1`.
- `package_node.ps1` – compacta o diretório `quantum_node` em um arquivo `.zip` para distribuição.
- `dna_events.jsonl` – arquivo de log em formato JSON Lines onde cada execução é registrada.

### 1. Cadastro e token na IBM Quantum
O site da IBM Quantum não está acessível a partir deste ambiente (“Site Unavailable”)【824281134064126†screenshot】. Portanto, você precisará abrir <https://quantum.ibm.com> no seu navegador local, criar uma conta (ou fazer login) e copiar o token da sua página de API. 

Após obter o token, exporte-o em sua sessão do PowerShell antes de executar os scripts:

```powershell
$env:IBMQ_TOKEN = "SEU_TOKEN_AQUI"
```

Se preferir usar um backend real (por exemplo `ibm_quito`, `ibm_oslo` etc.), substitua o valor de `-Backend` por um dos nomes listados na sua conta IBM. Para descobrir os dispositivos disponíveis, rode o código Python abaixo no seu ambiente local:

```python
from qiskit import IBMQ
IBMQ.load_account()
provider = IBMQ.get_provider(hub='ibm-q')
print([backend.name() for backend in provider.backends() if not backend.configuration().simulator])
```

### 2. Execução do nó
Para rodar um único nó com o simulador QASM:

```powershell
.\run.ps1 -NodeName "Teia Ajuda Viva Quântica" -Backend ibm_qasm_simulator
```

O script cria um circuito Bell (2 qubits), mede os bits, calcula a distribuição de contagens, classifica a entropia (Sub‑liminal, Vibrante, Caótico‑vivo ou Fractal‑Puro) e interpreta os bitstrings mais frequentes como rota fractal, modo de escuta e parâmetros do prompt TEIA. A saída é mostrada no console e o evento é acrescentado ao final de `dna_events.jsonl`.

### 3. Execução em lote
Para lançar vários nós de uma vez com sementes distintas:

```powershell
$names = "Nodo Alfa", "Nodo Beta", "Nodo Gama"
.\multi_run.ps1 -NodeNames $names -Backend ibm_qasm_simulator
```

Cada execução irá gerar uma semente aleatória e registrar o respectivo evento.

### 4. Interpretação simbiótica e entropia
O script `run.ps1` contém funções que interpretam os bitstrings:

- **Get‑FractalRoute** – usa os dois primeiros bits do resultado mais frequente para escolher um arquivo (ex. `fractal_index/linha_1.txt`, `linha_2.txt`, …).  
- **Get‑SymbioticMode** – mapeia outros dois bits para modos como *Eco*, *Ressonância*, *Sintonia* ou *Transcendência*.  
- **Get‑PromptParameters** – extrai pares de bits para definir `ModoParadoxador`, `Vibracao` e `Atencao`.  
- **Get‑SymbioticEntropyLevel** – calcula a entropia de Shannon e classifica em Sub‑liminal (🟡), Vibrante (🟢), Caótico‑vivo (🔴) ou Fractal‑Puro (🟣) de acordo com o nível de desordem.

Essas informações são exibidas no console e podem ser utilizadas por outras camadas da TEIA para ajustar o comportamento do nó.

.\package_node.ps1 -Output quantum_node_package.zip

O zip resultante incluirá todos os scripts e o `dna_events.jsonl`, facilitando a distribuição e implantação em outros ambientes.
