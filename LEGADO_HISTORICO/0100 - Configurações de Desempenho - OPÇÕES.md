FedCmMetricsEndpoint
Allows the FedCM API to send performance measurement to the metrics endpoint on the identity provider side. Requires FedCM to be enabled. – Mac, Windows, Linux, ChromeOS, Android, Fuchsia, Lacros

#fedcm-metrics-endpoint

Enable the systemEntropy property on PerformanceNavigationTiming
Allows developers to discern if the top level navigation occured during while the user agent was under load. See https://chromestatus.com/feature/5186950448283648 for more information. – Mac, Windows, Linux, ChromeOS, Android, Fuchsia, Lacros

#enable-system-entropy

Show performance metrics in HUD
Display the performance metrics of current page in a heads up display on the page. – Mac, Windows, Linux, ChromeOS, Android, Fuchsia, Lacros

#show-performance-metrics-hud

Preloading Settings on Performance Page
Moves preloading settings to the performance page. – Mac, Windows, Linux, ChromeOS, Fuchsia, Lacros

#preloading-on-performance-page

Choose ANGLE graphics backend
Choose the graphics backend for ANGLE. D3D11 is used on most Windows computers by default. Using the OpenGL driver as the graphics backend may result in higher performance in some graphics-heavy applications, particularly on NVIDIA GPUs. It can increase battery and memory usage of video playback. – Windows

#use-angle

Heap profiling sampling interval (in bytes).
Heap profiling service uses Poisson process to sample allocations. Default value for the interval between samples is 1000000 (1MB). This results in low noise for large and/or frequent allocations [size * frequency >> 1MB]. This means that aggregate numbers [e.g. total size of malloc-ed objects] and large and/or frequent allocations can be trusted with high fidelity. Lower intervals produce higher samples resolution, but come at a cost of higher performance overhead. – Mac, Windows, Linux, ChromeOS, Android, Fuchsia, Lacros

#memlog-sampling-rate

Enable (deprecated) synchronous mutation events
Mutation Events are a deprecated set of events which cause performance issues. Disabling this feature turns off Mutation Events. NOTE: Disabling these events can cause breakage on some sites that are still reliant on these deprecated features. – Mac, Windows, Linux, ChromeOS, Android, Fuchsia, Lacros

#mutation-events

HiDPI tab capture support for Cast Streaming
Enables HiDPI tab capture during Cast Streaming mirroring sessions. May reduce performance on some platforms and also improve quality of video frames. – Mac, Windows, Linux, ChromeOS, Fuchsia, Lacros

#enable-cast-streaming-with-hidpi

Toggle a performance metrics overlay while Cast Streaming
When enabled, a text overlay is rendered on top of each frame sent while Cast Streaming that includes frame duration, resolution, timestamp, low latency mode, capture duration, target playout delay, target bitrate, and encoder utilitization. – Mac, Windows, Linux, ChromeOS, Fuchsia, Lacros

#cast-streaming-performance-overlay

FedCmMetricsEndpoint
Allows the FedCM API to send performance measurement to the metrics endpoint on the identity provider side. Requires FedCM to be enabled. – Mac, Windows, Linux, ChromeOS, Android, Fuchsia, Lacros

#fedcm-metrics-endpoint

Compreendo sua necessidade de reduzir o uso de memória no Google Chrome. O Chrome pode ser conhecido por ser um navegador que consome muita memória, mas há algumas dicas que podem ajudar a reduzir esse consumo:

1. **Feche Abas Não Utilizadas:** Uma das principais razões para o alto consumo de memória no Chrome é a quantidade de abas abertas. Certifique-se de fechar abas que não estão sendo usadas no momento.

2. **Extensões:** Muitas extensões podem aumentar o uso de memória. Avalie quais extensões são essenciais para o seu trabalho e desative ou remova aquelas que não são necessárias.

3. **Utilize o Gerenciador de Tarefas do Chrome:** O Chrome possui um gerenciador de tarefas embutido. Pressione `Shift + Esc` enquanto estiver no Chrome para abrir o Gerenciador de Tarefas do Chrome. Isso permitirá que você veja quais guias e extensões estão consumindo mais memória e encerre aquelas que estão causando problemas.

4. **Ative a Suspensão de Abas:** O Chrome tem uma função chamada "Suspender Abas Inativas". Ela permite que o navegador suspenda automaticamente abas que não estão sendo usadas para liberar memória. Para ativá-la, vá em Configurações > Avançado > Sistema e ative a opção "Continuar executando aplicativos em segundo plano quando o Google Chrome estiver fechado."

5. **Atualize o Chrome:** Certifique-se de que você está usando a versão mais recente do Chrome, pois as atualizações geralmente incluem melhorias no gerenciamento de memória.

6. **Use Extensões de Gerenciamento de Memória:** Existem extensões como "The Great Suspender" que podem ajudar a suspender automaticamente guias inativas, economizando memória.

7. **Considere Outros Navegadores:** Em alguns casos, pode ser útil experimentar outros navegadores, como o Firefox, que tem um gerenciamento de memória diferente.

Lembre-se de que a quantidade de memória usada pelo Chrome também depende do conteúdo que você acessa. Páginas com muitos elementos gráficos ou scripts pesados podem consumir mais memória. Portanto, otimizar seu uso de memória também envolve escolher cuidadosamente os sites que você visita.

Espero que essas dicas ajudem a reduzir o uso de memória no seu navegador!
